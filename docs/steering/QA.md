## Test Strategy

ITA relies exclusively on Playwright E2E tests running against a real deployed environment. There are no unit or integration tests. All feature services implement interfaces for testability, but this capability is not yet exercised by dedicated unit test projects.

| Level       | Type              | Tool                        | Purpose                                                  |
| ----------- | ----------------- | --------------------------- | -------------------------------------------------------- |
| E2E         | User flows        | Playwright (.NET) + MSTest  | Validate full user journeys against real APIs and auth   |
| Lint/Format | Static analysis   | ESLint + Prettier           | Enforce code style and catch syntax issues               |
| Unit        | ‚Äî                 | ‚Äî                           | Not yet implemented                                      |
| Integration | ‚Äî                 | ‚Äî                           | Not yet implemented                                      |

## Coverage Requirements

No coverage thresholds are enforced ‚Äî only E2E tests exist and no coverage tooling is configured.

- **Every Task's Gherkin scenarios must be covered by at least one E2E test**
- **Bug fixes:** must include a regression E2E test that would have caught the bug
- **Role-based UI changes:** must have E2E assertions for both allowed and denied roles

## Test Patterns

- **Framework:** Playwright for .NET (`Microsoft.Playwright.MSTest` v1.55.0) with MSTest runner
- **Test file location:** `InterneTaakAfhandeling.EndToEndTest/{Feature}/{Feature}Scenarios.cs`
- **Naming:** Test classes `{Feature}Scenarios`, methods `User_CanDoX_FromY`, decorated with `[TestMethod("Human-readable description")]`
- **Base class:** All tests inherit `ITAPlaywrightTest` ‚Üí `PageTest`
- **Page objects:** Extension methods on `IPage` in `Locators.cs` / `BeheerLocators.cs`
- **Steps:** `await Step("description")` pattern for readable test output
- **Test data:** Created via real API calls (`TestDataHelper`) before tests; cleaned via `RegisterCleanup()` callbacks
- **Authentication:** Full Azure AD login with TOTP 2FA per test (real credentials via user-secrets or env vars)
- **Execution:** Serial (`[DoNotParallelize]`) ‚Äî no parallel test execution
- **Mock boundary:** No mocks ‚Äî all tests hit real external APIs. Keep it this way.
- **Database:** Not applicable ‚Äî ITA uses external APIs as source of truth

## Gherkin ‚Üí Test Mapping

Every Gherkin scenario from a Task issue must map to at least one Playwright E2E test. This is non-negotiable.

- Each Gherkin scenario maps to at least one `[TestMethod]` in the corresponding `{Feature}Scenarios.cs`
- Test method descriptions must reference the scenario name so failures are traceable
- Given steps ‚Üí test setup (API calls via `TestDataHelper`, navigation)
- When steps ‚Üí user action (Playwright interactions)
- Then steps ‚Üí assertion (`Expect(...).ToBeVisibleAsync()`, `Expect(...).Not.ToBeVisibleAsync()`)
- Scenarios describe observable behavior, not implementation details
- **Role-based scenarios** require separate test methods: one logged in with the role, one without

## Definition of Done

- [ ] All Gherkin scenarios from the Task issue are covered by automated E2E tests
- [ ] Linting passes (`npm run lint:ci`)
- [ ] Formatting passes (`npm run format:ci`)
- [ ] No new flaky tests introduced
- [ ] Regression test added for any bug fix
- [ ] Edge cases listed in the Feature are covered by E2E tests
- [ ] PR approved by at least one reviewer

## Test Commands

```bash
# Lint (CI mode ‚Äî no auto-fix)
cd InterneTaakAfhandeling.Web.Client && npm run lint:ci

# Format check (CI mode ‚Äî no writes)
cd InterneTaakAfhandeling.Web.Client && npm run format:ci
```

## Known Flaky Areas

- **Session timeouts** ‚Äî Authentication sessions expire during long test runs, causing intermittent failures. Mitigation: added time delays, but root cause (session lifetime vs test duration) is unresolved.
- **Logbook DOM changes** ‚Äî Large block of logbook verification code commented out due to DOM structure changes. Not yet re-enabled.
